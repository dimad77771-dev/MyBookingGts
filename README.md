# My Booking GTS

Первая безопасная версия робота EWRS.

## Что она делает

- запускает отдельный Microsoft Edge с CDP-портом и отдельным профилем;
- при повторном запуске подключается только к ранее запущенному управляемому Edge;
- не вводит логин, пароль и MFA, а ждёт ручной авторизации;
- каждый цикл начинает с `My bookings` и читает все страницы списка;
- любую найденную дату считает уже забронированной;
- пропускает сегодня, выходные, исключённые даты/интервалы и disabled-даты календаря;
- проверяет три ближайшие разрешённые даты;
- ищет столы по приоритету `W144`, `W146`, `W147`;
- при первом найденном приоритетном столе сохраняет screenshot + HTML + описание и останавливается **до нажатия любой кнопки бронирования**;
- если ничего не найдено, ждёт случайные 15–20 минут;
- пишет лог в консоль и `C:\tmp\mybooking_robot\log.txt`.

## Подготовка

1. Установить .NET 8 SDK.
2. Проверить `appsettings.json`, особенно путь к `msedge.exe`.
3. При необходимости добавить исключённые даты и интервалы в `appsettings.json`.
4. Выполнить:

```bat
dotnet restore
dotnet run --project MyBookingGts.csproj
```

Либо запустить `run.cmd`.

Playwright запускает установленный Edge самостоятельно через CDP, поэтому отдельный Chromium для этой программы не используется.

## Важное ограничение первой версии

Структура страницы с доступными столами нам пока неизвестна. При обнаружении текста приоритетного стола программа специально не нажимает ничего. Диагностика появится в:

```text
C:\tmp\mybooking_robot\errors\<timestamp>_preferred-desk-found\
```

После этого нужны screenshot и HTML, чтобы добавить реальный этап подтверждения бронирования.

## Остановка

Нажать `Ctrl+C` в консоли. Управляемый Edge намеренно не закрывается.


Пример исключений:

```json
"ExcludedDates": ["2026-08-17"],
"ExcludedDateRanges": [
  {
    "From": "2026-12-24",
    "To": "2027-01-04",
    "Reason": "Vacation"
  }
]
```
